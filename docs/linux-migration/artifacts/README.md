This directory contains the evidence artifacts generated during the Debian 13 workstation migration and hardening process. Each file corresponds to a step documented in the project overview and supports auditability, reproducibility, and technical verification.

---

## Installation & Verification Artifacts

**install_log.txt**  
Debian installer logs exported from `/var/log/installer/`, confirming successful installation and disk setup.

**hash_verification.txt**  
SHA-256 checksum output verifying the authenticity of the Debian 13 ISO.

**dmesg_init.txt**  
First-boot kernel log showing hardware initialization with no driver errors.

**apt_first_run.log**  
Output from the initial `apt update` and `apt full-upgrade -y`, establishing the patched system baseline.

---

## Hardening Artifacts

**nftables.conf**  
Active firewall ruleset enforcing deny-by-default filtering and SSH isolation.

**clamav_scan.log**  
Results of the ClamAV signature update and system scan, verifying operational antivirus tooling.

**ssh_daemon_state.txt**  
Service status, listening ports, and nftables verification demonstrating SSH is restricted to localhost only.

**sysctl_hardening.txt**  
Kernel parameters related to IPv6, ICMP, and system protections, including adjustments recommended by Lynis.

---

## Automation Suite Artifacts

**systemd_timer_list.txt**  
Snapshot of all active systemd timers, confirming the automated maintenance schedule is enabled.

**systemd/**  
Includes all custom service and timer units for daily updates, log cleanup, disk monitoring, weekly audits, and monthly archiving.

**maintenance_logs/**  
Log outputs generated by the automation suite, stored under `/var/log/maintenance/`.

**final_system_audit.txt**  
Combined Lynis system audit and ClamAV scan summary showing security controls and audit results.

**metrics_summary.md**  
CPU, RAM, uptime, kernel, and filesystem usage metrics for the hardened workstation.

**nftables_final.conf**  
Version-controlled final firewall configuration matching the deployed ruleset.

---

## Notes

- All artifacts are provided in raw form for transparency.  
- Logs may contain timestamps or counters from the moment of capture.  
- No artifact has been edited beyond necessary sanitization.  

These artifacts collectively provide a full, verifiable trail of the Debian workstation migration and hardening process.
